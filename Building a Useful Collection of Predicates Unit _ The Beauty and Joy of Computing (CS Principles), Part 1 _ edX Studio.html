    Building a useful collection of predicates

    Some tests—for example, a test to see if a number is even or if a letter is a vowel—are needed so often that you won't want to re-create them each time you need them. In this activity, you will create a set of tools that you may use often, especially for complicated programming tasks.

    You've already built a predicate vowel? to test for vowels. In the same lesson, you built a block that says whether a number is even or odd, but not a predicate that you could use inside if to choose different actions depending on whether a number is even or odd. Snap! has a built-in test to see if something is a number, but not a test to see if the number is an integer, and often you will want that.

    1. Create the predicate to report true when its input it even and false otherwise.
